a.

An analytics company might be concerned about browser caching because in order to accurately count the number of times a user visits a site, because caching will save the count and won't update it appropriately with a new visit or page refresh.

b. Prevent Browser Caching

I had to look this up, and based on a number of resources including StackOverflow, Medium, and a Google search, it seems it can be accomplished by adding tags (ETags and Cache control) to the HTML header.

Those tags look like this: 

    
        <meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate"/>
        <meta http-equiv="Pragma" content="no-cache"/>
        <meta http-equiv="Expires" content="0"/>

c.

Some browsers will only load HTTPS, or secure pages. The solution here is to use a javascript tracking pixel, rather than an img tag--the img tag might reference non-secure assets and will be blocked by the client, but a javascript tag will still load if the overall page is secure.

d.

Companies can collect information using a tracking pixel:

  • IP address
  • Browser/email client used
  • Mobile or desktop
  • Operating System used
  • Time opened/accessed
